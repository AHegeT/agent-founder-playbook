# Meetup & Playbook Workflow Guide

## Complete Workflow

The AI Business Roundtable workflow consists of four main phases:

### Phase 1: Planning & Announcement
1. Decide on session topic and date
2. Generate announcements for all platforms
3. Post to Meetup, WhatsApp, LinkedIn, and Luma
4. Monitor RSVPs and engagement

### Phase 2: Session Preparation
1. Review topic and prepare discussion points
2. Send reminder announcements (1 week before, 1 day before)
3. Prepare session materials if needed

### Phase 3: Running the Session
1. Welcome attendees
2. Facilitate roundtable discussion
3. Take notes on key insights and case studies
4. Document action items and resources mentioned

### Phase 4: Post-Session Playbook
1. Debrief with core members (capture fresh insights)
2. Generate playbook template
3. Fill in all sections with notes from debrief
4. Review for clarity and completeness
5. Publish to website
6. Share playbook link with attendees

## Announcement Distribution Strategy

### Meetup.com (Primary Platform)
- Post full detailed announcement
- Include all session details
- Enable RSVP tracking
- This becomes the "source of truth" link

### WhatsApp Groups
- Post brief announcement with Meetup link
- Target different groups with slight variations in intro
- Keep it casual and conversational
- Include emojis for engagement

### LinkedIn
- Professional tone
- Emphasize business value and networking
- Include relevant hashtags
- Encourage engagement (comments/DMs)
- Tag relevant people if appropriate

### Luma
- Event-focused description
- Highlight unique format and value
- Include post-session deliverable (playbook)
- Professional but approachable tone

## Playbook Structure Guidelines

Each playbook should follow this structure:

### Frontmatter (YAML)
- Title (session topic)
- Session number
- Date
- Location
- Number of attendees
- Tags (for website categorization)

### Content Sections

**Session Overview**
- 2-3 paragraphs setting context
- What was the main theme?
- What questions did we explore?

**Key Insights** (3-5 insights)
- Each insight as a heading
- Context and explanation
- Why it matters

**Action Items**
- Specific, actionable steps
- Can be formatted as checkboxes
- Relevant for different audience types

**Resources Discussed**
- Tools and platforms mentioned
- Articles and documentation shared
- Links to previous related playbooks

**Case Studies & Examples**
- Real implementations discussed
- Challenge → Solution → Results format
- Key lessons learned

**Key Takeaways**
- 3-5 main points
- Digestible summary format
- What readers should remember

## Tips for Quality Playbooks

1. **Write for your future self** - Will this be useful in 6 months?
2. **Be specific** - Avoid generic advice, include actual examples
3. **Link resources** - Make it easy to dive deeper
4. **Use real names/companies** (with permission) - Authenticity matters
5. **Include contrarian views** - Document disagreements and trade-offs
6. **Make it scannable** - Use headings, bullets, and formatting
7. **Publish quickly** - Ship within 3-5 days while memory is fresh

## Common Variations

### Different Group Types (WhatsApp)
- **Tech founders group**: Emphasize technical depth
- **General entrepreneurs**: Focus on business value
- **Local community**: Highlight networking and location

### Seasonal Adjustments
- **Start of year**: Focus on planning and strategy
- **Mid-year**: Implementation and case studies
- **End of year**: Reflection and looking ahead

### Advanced vs Beginner Topics
- **Beginner-friendly**: Include more context and resources
- **Advanced**: Assume knowledge, focus on edge cases
